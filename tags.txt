We can use the last 3 letters to determine what kind of point we're dealing with.
If a point ends with "SEN", then we know it's essentially a 'read only' point.
If a point ends with "CMD", then we know we can write to it (it's some sort of set point or
input point that we can use to control it), *but* we can also read from that point to get the
current value of it

For objects, we require they have dictionaries of the following:
  required setpoints/writeables: for an airhandler, this would be the zone air temperature setpoint.
                            basically stuff that is a higher level attribute of the holistic
                            air handler
  requried points: for an air handler, this would be everything in test.py (assuming that's
                  the minimum set of points

for points, we require they have dictionaries of the following:
  required setpoints: writeables
  required attributes: readables

It's important to maintain
the following structure:
object (node in graph)
|
point (node in graph)
|
attributes (key in dict)
